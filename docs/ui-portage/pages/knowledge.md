# Knowledge (hors mockup, style dérivé strict)

## Structure des zones
- Hero knowledge.
- KPI snippets.
- Barre recherche + filtres.
- Grille snippets/cases.
- CTA réutilisation.

## Composants UI utilisés
- `Card`, `Btn`, `Input`, `Empty`, `Spinner`, `Icon`.

## Features
- Bibliothèque de snippets objections.
- Filtres type/qualité/recherche.
- Copie rapide script.
- Lien vers debrief contextuel.

## États
- loading data.
- empty global.
- empty filtres.
- erreurs API objections.

## Interactions
- recherche live.
- changement filtres.
- copier snippet.
- ouvrir debrief source.

## Responsive attendu
- filtres empilés mobile.
- cards en colonne unique mobile.

## Écarts mockup vs app actuelle
- Page non couverte explicitement par mockup.
- Portage style strict dérivé requis.

## Critères d’acceptation parité visuelle
- Même qualité de contraste et rythme que pages mockup.
- lisibilité texte longue maintenue.
