# Team (HOS/Admin)

## Structure des zones
- Hero page équipe.
- Sélecteur équipe(s) + KPIs équipe.
- Radar/barres comparatives.
- Liste membres (cards expansibles).
- Actions objectifs/invitations/gestion membres.

## Composants UI utilisés
- `Card`, `Btn`, `Input`, `Textarea`, `Spinner`, `Empty`, `Icon`.
- `TeamTile`, `MemberCard`, `ManagerCopilotCard`, `Chart`.

## Features
- CRUD équipes.
- Gestion membres (déplacer/retirer).
- Codes invitation.
- Objectifs et plans d’action.
- Résumé manager IA.

## États
- loading équipes/membres.
- empty sans équipe.
- erreurs API équipe/summary.
- succès actions membres/objectifs.

## Interactions
- sélection équipe.
- expansion membre.
- assignation objectifs.
- gestion invitations.

## Responsive attendu
- Desktop: vue comparative dense.
- Mobile: sections empilées, actions accessibles.

## Écarts mockup vs app actuelle
- Certaines cartes analytiques sont plus techniques que le mockup.

## Critères d’acceptation parité visuelle
- Rendu managérial clair, sans surcharge.
- Actions critiques visibles en priorité.
- Cohérence visuelle avec dashboard manager.
